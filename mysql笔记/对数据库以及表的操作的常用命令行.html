  • 主键约束：

    primary key

    这个命令是给数据库中的表中的某个字段添加主键约束，主键常用作我们获取到的数据的id。因为主键表示在数据库中的某个表里的某个字段的值不能重复且不能为null。

    我们可以在创建表的时候去给表中的某个字段设置主键

    我们在去创建表时是这样的：CREATE TABLE 要创建表的名字 (字段1的名字 字段1的类型，字短2的名字 字段2的类型,...,字段n的名字 字段n的类型);  我们可以给某个字段的字段类型后边去加上设置主键的命令那么这个地段就会成为主键。

    我们还可以在去创建表的时候这样设置主键

    我们在去创建表时是这样的：CREATE TABLE 要创建表的名字 (字段1的名字 字段1的类型，字短2的名字 字段2的类型,...,字段n的名字 字段n的类型，primary key（字段名）);  这样primary key（字段名）里的字段就会成为主键，括号里可以写多个字段名用逗号隔开这样就会成为设置复合主键。

    我们还可以在去修改表的时候去添加一个主键

    alter table 表名 add primary key(字段名)这样我们就给某个字段添加了主键

  • 外键约束：

    constraint 外键名 foreign key (字段名) references 练接的表名 (连接表的字段)

    外键约束主要用于两个表之间的连接比如有a和b两个表我们通过a表的外键去查询b表的主键然后找到我们所需要的数据

    在创建表时添加外键约束

    我们在去创建表时是这样的：CREATE TABLE 要创建表的名字 (字段1的名字 字段1的类型，字短2的名字 字段2的类型,...,字段n的名字 字段n的类型,constraint 外键名 foreign key (字段名) references 练接的表名 (连接表的字段该字段必须为主键));  比如当前的表名为a，连接的表名为b，那么我们就给a中的某个字段设置了外键我们可以通过外键去查询b表中的主键来查询到我们需要的数据

    我们还可以在去修改表的时候去添加一个外键

    alter table 表名 add constraint 外键名 foreign key (字段名) references 练接的表名 (连接表的字段));

    我们还可以在去修改表的时候去删除外键

    alter table 表名 drop foreign key 外键名;

  • 唯一约束：

    unique

    唯一约束主要是帮助我们在去创建某个字段的时候这个字段里的数据必须都是唯一的不能有重复但是可以有空值,注:可能会对唯一约束与主键约束有些混淆,但是一个表中可以有多个唯一字段为唯一约束但是主键约束只能有一个且主键约束的字段里不能有空之唯一约束可以

    我们可以在去创建表的时候去添加唯一约束:

    我们在去创建表时是这样的:create table 表名 (字段1的名字 字段1的类型，字短2的名字 字段2的类型,...,字段n的名字 字段n的类型 unique)   像这样我们给某个字段的字段类型后加上unique就能给该字段添加上唯一约束

    我们还可以在修改表时去添加唯一约束:

    alter table 表名 add constraint 唯一约束名 unique(要添加唯一约束的字段名)

    删除唯一约束:

    alter table 表名 drop index 字段名

  • 检查约束:

    check (表达式)

    检查约束主要用于检查我们的表中的某些数据满足表达式的比如id>1;

    我们可以在创建表时添加检查约束

    create table 表名 (字段1 字段类型,字段2 字段类型,...,字段n 字段类型,check (对某个字段的约束比如id>1));  这样我们就给叫做id的字段添加了约束这个叫做id的字段下边所有的值不能小于1

    我们还可以在修改表时添加检查约束:

    alter table 表名 add constraint 检查约束名 check (表达式);

    删除检查约束:

    alter table 表名 drop constraint 检查约束名

  • 默认值约束:

    default 默认值

    默认值就是当我们再去添加数据时没有给某个字段赋值或者它的值为空那么它的值将为默认值.

    我们在去创建表的时候添加默认值:

    create table 表名 (字段1 字段类型 default 默认值,字段2 字段类型,...,字段n 字段类型);  像这样我们给字段1添加了默认值如果字段1的某条数据的值为空的话那么它的值就是默认值.

    我们可以在去修改表的时候添加默认值:

    alter table 表名 change column 字段名 数据类型 dafalut 默认值

    删除默认值约束:

    alter table 表名 change column 字段名 字段名 数据类型 default null;

  • 非空约束:

    not null

    非空约束就是我们在去插入数据的时候如果我们给某个字段设置了非空约束那么这个字段里的数据就不能为空如果为空则会报错.

    在创建表时区设置非空约束

    create table 表名 (字段1 数据类型,字段2 数据类型 not null)  像这样给某个字段的苏剧类型后加上NOT NULL 那么该字段的数据就不能有空值

    在修改表时设置非空约束:

    ALTER TABLE 数据表名 cHANGE COLUMN 字段名 字段名 数据类型 NOT NULL;

    删除非空约束:

    ALTER TABLE 表名 CHANGE COLUMN 字段名 字段名 数据类型 NULL;

  • 查看表中所有的约束:

    show create table 表名

  • 查询语句:

    • select * from 表名

      这条语句主要是用来查询表中所有的数据

    • select 字段名 from 表名

    • SELECT DISTINCT 字段名 FROM 表名;

    • select * from 表名 limit a ,b

    • select * from 表名 where 限制条件

    • insert into 表名 (字段1,字段2) values (数据1,数据2)

    删除数据:

    • delete from 表名 where 限制条件

      这条命令主要用于删除表中的数据一定结合限制条件去使用否则会删除表中所有的数据.

视图:

什么是视图?

视图简单来说就是一条select语句进行查询后的一个结果集,当我们去对视图中的数据进行修改时那么原表中的数据也会跟着修改,当我们去修改原表中的内容时视图中的内容也会跟着更改

  • create view 视图名 as select查询语句

    这条语句主要用来创建视图

    create view 视图名 (字段名1,字段名2,...,字段名n) as select 字段名1,字段名2 from 表名

    有时候我们会使用自己指定的字段名那么我们的视图中就会是我们自己指定的字段名然后数据还是表中的数据

  • describe 视图名

    我们通过这个命令可以查询到视图

  • alter view 视图名 as select语句

    这个命令用来修改视图

    update 视图名 set 字段名=新值

    这个命令也是用来修改视图中的命令但是也一定要结合where 限制条件去使用 否则将会修改整个数据

  • drop view 视图名

    使用这个命令能够进行删除视图

存储过程:

什么是存储过程?

上面的sql语句等都是对单个表的操作而存储过程是为了处理多条sql语句而存在的操作,使用存储过程的目的是将常用或者复杂的工作的sql语句先实现存储起来在数据库服务器中,当我们需要完成与存储过程相同的操作时只需 call存储过程

  • delimter 结束字符

    修改结束字符，因为我们的sql语句是默认以;结尾的但是当我们在去创建存储过程时我们过程体可能会包含多条sql语句如果这些sql语句都以;进行结尾那么我们的程序会默认只执行第一条sql语句，所以我们要利用上面的命令行将结束符修改，sql语句仍用;结尾。

  • create procedure 存储过程名(参数 参数类型) begin sql语句 end

    创建存储过程

  • 修改存储过程，修改存储过程的内容可以先删除原存储过程，再以相同的命名创建新的存储过程。修改存储过程的名称可以通过删除原存储过程再以不同的命名创建新的存储过程。

  • drop procedure 存储过程名

    删除存储过程

触发器：

什么是触发器?

触发器就是一种特殊的存储过程，不同的是存储过程需要call调用才能够使用，而触发器不需要，触发器有两种类型：before 和after类型，before是在激活他的语句之前启动触发器，after是在激活他的语句之后启动触发器，启动触发器的语句有三种分别是insert delete undate

  • create trigger 触发器名 触发器类型 启动触发器的是三种语句的哪个 on 表名 for each row 启动触发器时要执行的sql语句;

    创建触发器

  • drop TRIGGER 触发器名

    删除触发器，不存在修改触发器如果要修改触发器可通过删除触发器然后再以相同的名字命名

索引就相当于是表的目录，我们在查找内容前可以先查看目录再去根据目录去进行对数据的查询

  • create table 表名 (字段1 数据类型,字段2 数据类型,...,字段n 数据类型,index(字段名));

    这个命令用来创建一般索引

    create table 表名 (字段1 数据类型,字段2 数据类型,...,字段n 数据类型,unique index(字段名))

    这个命令用来创建唯一索引

  • show index from 表名

    该命令用于查看表中的索引

  • drop index 索引名 on index 表名

    删除索引

事务：

事务是最小的工作单位不可再分，事务只和insert ，update ，delete 语句也就是插入，修改，删除数据有关，也就是说只有这三种语句才有事务，当执行这三种语句的某一个就代表自动开启了一个事务，这三个语句执行的结束也就自动关闭了事务

begin

这个命令表示手动开启了一个事务

commit

这个命令表示事务的提交

rollback

这个命令表示事务的回滚，事务的回滚表示重新回到事务的开始然后重新输入sql语句来进行事务
